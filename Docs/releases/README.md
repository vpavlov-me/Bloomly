# Release Notes

Store release notes in `Docs/releases/<version>.md`. Recommended structure:

```markdown
# BabyTrack <version> — <date>

## 🚀 Highlights
- ...

## 🐛 Fixes
- ...

## 🔧 Tech
- ...

## 📦 Migration
- Data migration steps, if required.
```

Add a link to the corresponding file when publishing the GitHub Release.
